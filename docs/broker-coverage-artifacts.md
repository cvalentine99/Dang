# Broker Coverage Enrichment Artifacts

The `/admin/broker-coverage` admin cockpit uses two generated JSON artifacts
to enrich its analysis with frontend callsite data and param parity details.

## Required Artifacts

| File | Generated by | Purpose |
|------|-------------|---------|
| `docs/wiring-ledger.json` | `scripts/generate-wiring-ledger.mjs` | Maps each tRPC procedure to its frontend callsites (file + line) |
| `docs/ui-param-parity.json` | `scripts/audit-ui-param-parity.mjs` | Records which query params each callsite actually passes to each procedure |

## How to Generate

```bash
# Generate wiring ledger (scans router + client source)
node scripts/generate-wiring-ledger.mjs

# Generate param parity audit (scans all trpc.wazuh.* callsites)
node scripts/audit-ui-param-parity.mjs
```

Both scripts are deterministic, read-only source scans. They produce
`docs/wiring-ledger.json` and `docs/ui-param-parity.json` respectively.

Run them after any changes to:
- `server/wazuh/wazuhRouter.ts` (procedure additions/removals)
- `client/src/pages/*.tsx` (new callsites or param changes)

## What Degrades Without Them

The broker coverage page **does not crash** if artifacts are missing.
It degrades gracefully:

| Artifact Missing | Effect |
|-----------------|--------|
| `wiring-ledger.json` | Callsite column shows 0 for all endpoints. Drill-through and route inference still work structurally but have no data. |
| `ui-param-parity.json` | Parity section in the detail drawer is empty. Parity summary shows "No parity observed." |
| Both missing | An amber warning banner appears: "Wiring ledger and parity artifacts not loaded." Coverage stats, wiring levels, and broker config data remain fully accurate (they are computed from source, not artifacts). |

The `enrichment` field in the API response reports load status:
```json
{
  "enrichment": {
    "wiringLedgerLoaded": true,
    "parityArtifactLoaded": true
  }
}
```

## CI / Build Validation

Use `scripts/check-broker-artifacts.sh` to warn if artifacts are absent
or stale. This is advisory — it exits 0 with warnings, not a hard failure.
